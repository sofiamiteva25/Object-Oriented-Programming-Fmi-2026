# Задачи за практикум — Умни указатели

---

## Задача 1 — Система за доставки (`std::unique_ptr`)

Разработваш система за управление на пратки в логистична компания. Всяка пратка се зачислява към **точно един склад** — тя не може да съществува в два склада едновременно. Когато пратката се прехвърли в друг склад, тя напуска предишния. Когато склад се закрие, всички пратки в него се унищожават.

---

### Клас `Package`

Всяка пратка се характеризира с:
- Трекинг номер — уникален идентификатор с произволна дължина (помислете за добър начин за гарантиране на уникалността на низовете)
- Получател с произволна дължина
- Тегло в килограми (`double`)
- Приоритет: `1` (нормален), `2` (експресен), `3` (спешен)

**Класът няма конструктор по подразбиране.**

Функционалности:
- Пратките да могат да се **извеждат** на стандартния изход.
- Две пратки да могат да се **сравняват** — първо по приоритет (по-висок е „по-голям"), при равен приоритет — по тегло.
- `getTrackingNumber()`, `getWeight()`, `getPriority()` — getter-и.

---

### Клас `Warehouse` (Склад)

Складът има наименование, капацитет (максимален брой пратки) и **притежава** своите пратки. Пратката принадлежи на точно един склад — изберете структура за управление на паметта, която отразява тази **собственост** и гарантира, че указателите към пратките остават валидни при добавяне на нови.

**Функционалности:**

- Конструктор, приемащ наименование и капацитет.
- `addPackage(trackingNum, recipient, weight, priority)` — добавя нова пратка. При вече съществуващ трекинг номер — игнорира се. При запълнен капацитет — игнорира се.
- `removePackage(trackingNum)` — премахва пратка по трекинг номер.
- `operator[](trackingNum)` — достъп до пратка по трекинг номер. Връща указател към пратката или `nullptr` ако не е намерена.
- `transferTo(trackingNum, other)` — **прехвърля** пратка от текущия склад в `other`. При ненамерен трекинг номер или запълнен целеви склад — операцията се игнорира. Пратката трябва да **напусне** текущия склад и да **влезе** в другия — без копиране.
- `heaviestPackage()` — връща указател към най-тежката пратка, `nullptr` при празен склад.
- `urgentPackages()` — връща брой пратки с приоритет `3` (спешни).
- `totalWeight()` — връща общото тегло на всички пратки в склада.
- `operator<<` — извежда склада: наименование, капацитет, пратки наредени по приоритет (най-спешните първи).
- `operator bool` — складът е „претоварен" ако е запълнен над 80% от капацитета си.
- `operator()` — приема трекинг номер и нов приоритет, обновява приоритета на пратката ако е намерена. Връща референция към текущия склад за верижно извикване.
- Класът спазва **Rule of Zero**.

---

### Пример за използване

```cpp
int main() {
    Warehouse sofia("София", 10);
    Warehouse plovdiv("Пловдив", 5);

    sofia.addPackage("BG001", "Иван Иванов",  2.5, 1);
    sofia.addPackage("BG002", "Мария Петрова", 0.8, 3);
    sofia.addPackage("BG003", "Петър Георгиев",5.1, 2);

    std::cout << sofia << "\n";

    // Прехвърляне — BG003 напуска София и влиза в Пловдив:
    sofia.transferTo("BG003", plovdiv);

    if (Package* p = sofia["BG001"])
        std::cout << "Намерена: " << *p << "\n";

    if (Package* p = sofia.heaviestPackage())
        std::cout << "Най-тежка: " << *p << "\n";

    std::cout << "Спешни пратки: " << sofia.urgentPackages() << "\n";
    std::cout << "Общо тегло: "    << sofia.totalWeight()    << " кг.\n";

    // Верижно обновяване на приоритет:
    sofia("BG001", 3)("BG002", 2);

    if (sofia)
        std::cout << "Складът е претоварен!\n";
    else
        std::cout << "Складът работи нормално.\n";

    std::cout << plovdiv << "\n";
}
```

## Задача 2 — Изследователска мрежа (`std::shared_ptr`)

### Контекст

Разработваш система за управление на научни изследователи и проекти. Един изследовател може да участва в **множество проекти едновременно** — например проф. Иванов ръководи едновременно проект по AI и проект по Роботика. Когато проект приключи, изследователите **не изчезват** — те продължават да участват в другите си проекти. Изследователят се унищожава едва когато **не участва в нито един активен проект**.

---

### Клас `Researcher`

Всеки изследовател се характеризира с:
- Три имена с произволна дължина
- Идентификатор (ID) — уникален
- Специализация с произволна дължина
- Брой публикации

**Класът няма конструктор по подразбиране.**

Функционалности:
- Изследователите да могат да се **извеждат** на стандартния изход.
- Двама изследователи да могат да се **сравняват** — по брой публикации. При равен брой — лексикографски по специализация.
- `getId()`, `getPublications()` — getter-и.
- `addPublication()` — увеличава броя публикации с 1.

---

### Клас `ResearchProject` (Изследователски проект)

Проектът има наименование, бюджет (`double`) и **споделена собственост** върху изследователите — един изследовател може да участва в повече от един проект едновременно. Изберете структура за управление на паметта, която отразява тази **споделена собственост**.

**Функционалности:**

- Конструктор, приемащ наименование и бюджет.
- `addResearcher(researcher)` — добавя изследовател към проекта. При вече съществуващ ID — игнорира се.
- `removeResearcher(id)` — премахва изследовател от проекта. Изследователят **не се унищожава** ако участва в други проекти.
- `findResearcher(id)` — търси изследовател по ID. Ако не е намерен — не е грешка, просто го няма.
- `leadResearcher()` — връща изследователя с най-много публикации. Ако проектът е празен — семантично правилното е „няма такъв".
- `publishAll()` — добавя по една публикация на **всички** изследователи в проекта. Тъй като изследователите са споделени — публикациите им се отразяват и в другите проекти, в които участват.
- `totalBudgetPerResearcher()` — връща бюджета разделен на броя изследователи. При празен проект — връща 0.
- `operator<<` — извежда проекта: наименование, бюджет, изследователи наредени по брой публикации (най-много първи).
- `operator bool` — проектът е „финансово стабилен" ако бюджетът на изследовател надвишава 10 000.
- `operator()` — приема ID и нова специализация, обновява специализацията на изследователя ако е намерен. Тъй като изследователят е споделен — промяната се вижда и в другите проекти. Връща референция към текущия проект за верижно извикване.
- Класът спазва **Rule of Zero**.

---

### Пример за използване

```cpp
int main() {
    // Изследователите се създават веднъж и се споделят:
    auto ivanov  = /* създаване на Researcher — проф. Иванов, AI, 45 публикации */;
    auto petrova = /* създаване на Researcher — доц. Петрова, Роботика, 32 публикации */;
    auto georgiev= /* създаване на Researcher — д-р Георгиев, ML, 18 публикации */;

    ResearchProject ai("Проект AI", 150000.0);
    ResearchProject robotics("Проект Роботика", 80000.0);

    ai.addResearcher(ivanov);
    ai.addResearcher(petrova);
    ai.addResearcher(georgiev);

    robotics.addResearcher(ivanov);    // Иванов участва и в двата проекта
    robotics.addResearcher(petrova);   // Петрова също

    std::cout << ai       << "\n";
    std::cout << robotics << "\n";

    // publishAll в AI добавя публикации — видими и в Роботика:
    ai.publishAll();
    std::cout << "След publishAll в AI:\n" << robotics << "\n";
    // Иванов вече има 46 публикации и в двата проекта!

    if (auto lead = ai.leadResearcher())
        std::cout << "Водещ изследовател: " << *lead << "\n";

    if (auto r = robotics.findResearcher(georgiev->getId()))
        std::cout << "Намерен в Роботика: " << *r << "\n";
    else
        std::cout << "Георгиев не е в Роботика\n";  // защото не е добавян

    // Верижно обновяване на специализация — вижда се и в другия проект:
    ai(ivanov->getId(), "Deep Learning")(petrova->getId(), "Computer Vision");

    std::cout << "Бюджет на изследовател (AI): "
              << ai.totalBudgetPerResearcher() << " лв.\n";

    if (ai)       std::cout << "AI е финансово стабилен\n";
    if (robotics) std::cout << "Роботика е финансово стабилен\n";

    // Премахване от AI — Иванов продължава да е в Роботика:
    ai.removeResearcher(ivanov->getId());
    std::cout << "След премахване от AI:\n" << robotics << "\n";
}
```
