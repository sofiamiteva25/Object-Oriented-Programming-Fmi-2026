## 1. Какво е Голямата Четворка?

**Голямата Четворка** е набор от четири специални функции, които всеки клас в C++ притежава — или дефинирани от програмиста, или автоматично генерирани от компилатора:

| # | Функция | Кога се извиква |
|---|---|---|
| 1 | **Конструктор по подразбиране** | При създаване на обект без аргументи |
| 2 | **Копиращ конструктор** | При създаване на нов обект като копие на съществуващ |
| 3 | **Оператор `=`** | При присвояване на стойност на вече съществуващ обект |
| 4 | **Деструктор** | При унищожаване на обект |

---

За прост клас без динамична памет компилаторът генерира всичките четири автоматично и те работят коректно:

```cpp
class Test {
    char str[20];
    int  number;
};

int main() {
    Test obj1;          // конструктор по подразбиране
    Test obj2(obj1);    // копиращ конструктор
    obj1 = obj2;        // оператор=
}                       // деструктор (x2)
```

Кодът се компилира успешно и функциите имат правилно поведение — **но само докато в класа няма динамична памет**.

---

## 2. Как работят автоматично генерираните функции?

Всяка от четирите функции, когато е генерирана от компилатора, **извиква рекурсивно същата функция върху всяка член-данна** на класа.

### Конструктор по подразбиране

Извиква конструкторите по подразбиране на всички член-данни:

```
Test::Test()
├── char str[20] → инициализира масива
└── int number   → инициализира int-а
```

```cpp
class A {
public:
    A() { std::cout << "A конструктор\n"; }
};
class B {
public:
    B() { std::cout << "B конструктор\n"; }
};

class Test {
    char str[20];
    A    obj2;
    B    obj3;
};

Test t;
// Изход:
// A конструктор   ← извикан автоматично
// B конструктор   ← извикан автоматично
```

![constr](https://i.ibb.co/s2m8XtC/1.png)

### Копиращ конструктор

Извиква копиращите конструктори на всички член-данни:

```
Test::Test(const Test& other)
├── char str[20] → копира масива елемент по елемент
├── A obj2       → извиква A копиращ конструктор
└── B obj3       → извиква B копиращ конструктор
```

![copy-constr](https://i.ibb.co/9Vqk7Mn/3.png)

### Деструктор

Извиква деструкторите на всички член-данни в **обратен ред** спрямо конструкцията:

```
Test::~Test()
├── B obj3       → извиква ~B() (обратен ред)
├── A obj2       → извиква ~A()
└── char str[20] → унищожава масива
```

![destr](https://i.ibb.co/kmYSzP7/2.png)

### Важното заключение

За прости типове (`int`, `double`, масиви с фиксиран размер, обекти без динамична памет) тези автоматични операции са напълно коректни. Проблемът възниква при **указатели към динамично заделена памет**.

## 3. Проблемът — Shallow Copy

Разглеждаме клас с динамична памет:

```cpp
struct Person {
    Person(const char* name, int age) {
        this->name = new char[strlen(name) + 1];
        strcpy(this->name, name);
        this->age = age;
    }
private:
    char* name;
    int   age;
};

int main() {
    Person p1("Иван", 20);
    Person p2(p1);           // извиква се автоматично генерирания копиращ конструктор
}
```

Автоматично генерираният копиращ конструктор копира **стойността на указателя** — не данните, към които сочи. Резултатът е, че `p1.name` и `p2.name` сочат към **едно и също място в паметта**:

![enter image description here](https://i.ibb.co/q5rfGBf/Capture.png)

Това е **Shallow Copy** (плитко копиране). Последствията:

- Промяна на `p2.name` ще промени и `p1.name` — двата обекта са засегнати
- При унищожаване на `p2` паметта се освобождава
- При унищожаване на `p1` се прави `delete[]` върху вече освободена памет → **crash**

---

## 4. Решението — Deep Copy

**Deep Copy** (дълбоко копиране) заделя нова памет и копира съдържанието. Двата обекта получават **независими копия** на данните:

![enter image description here](https://i.ibb.co/XZq5rGT/33.png)

Промяна или унищожаване на единия обект не засяга другия по никакъв начин.

**Заключение:** При всеки клас с динамична памет трябва ръчно да се имплементират всичките четири функции от Голямата Четворка.

---

## 5. Конструктор по подразбиране

Конструкторът по подразбиране се извиква при създаване на обект **без аргументи**. Отговорността му е да постави обекта в **валидно начално състояние**.

```cpp
Person() : name(nullptr), age(0) {}
```

При динамична памет е критично указателите да се инициализират с `nullptr`. Неинициализиран указател съдържа произволна стойност — `delete[]` върху него е недефинирано поведение.

### Кога компилаторът НЕ генерира конструктор по подразбиране?

Ако програмистът е дефинирал **какъвто и да е друг конструктор**, компилаторът **спира** да генерира конструктор по подразбиране:

```cpp
class Person {
public:
    Person(const char* name, int age) { /* ... */ }
    // Компилаторът НЕ генерира Person() автоматично!
};

Person p;           // ГРЕШКА — няма конструктор по подразбиране
Person arr[10];     // ГРЕШКА — масивът изисква конструктор по подразбиране
```

Ако е нужен, трябва да се дефинира изрично.

---

## 6. Копиращ конструктор

### Дефиниция

Копиращият конструктор се извиква при **създаване на нов обект като копие на съществуващ**. Сигнатурата му е фиксирана — параметърът задължително е `const` референция:

```cpp
Person(const Person& other);
```

Ако параметърът беше по стойност, за подаването му щеше да е нужно копие → щеше да се извика копиращият конструктор → **безкраен цикъл**.

### Кога се извиква

```cpp
Person p1("Иван", 20);

Person p2 = p1;           // копиращ конструктор — p2 се създава
Person p3(p1);            // копиращ конструктор — p3 се създава

void print(Person p);
print(p1);                // копиращ конструктор — при подаване по стойност
```

### Имплементация

```cpp
Person(const Person& other) {
    copyFrom(other);   // делегираме към помощната функция
}
```

Детайлната имплементация на `copyFrom` е описана в [Секция 9](#9-помощните-функции-free-и-copyfrom).

## 7. Оператор= (Copy Assignment)

### Разлика от копиращия конструктор

```cpp
Person p1("Иван", 20);

Person p2 = p1;   // КОПИРАЩ КОНСТРУКТОР — p2 се създава тук

Person p3("Мария", 21);
p3 = p1;          // ОПЕРАТОР= — p3 вече съществуваше преди
```

Копиращият конструктор се извиква при **създаване** на обект. Операторът `=` се извиква когато обектът **вече съществува** и му се присвоява нова стойност.

### Сигнатура

```cpp
Person& operator=(const Person& other);
```

Операторът връща `Person&` за да поддържа верижно присвояване: `a = b = c`.

### Задължителни стъпки

```cpp
Person& operator=(const Person& other) {
    // Проверяваме дали other не сочи към същия обект като this
    // (например person1 = person1), защото тогава, ако изтрием данните,
    // не можем да копираме отново от същия обект.
    // В такъв случай просто не правим нищо и връщаме *this.
    if (this != &other) {
        free();           // зачистваме старата динамична памет преди копиране
        copyFrom(other);  // копираме
    }
    return *this;
}
```

Без проверката в стъпка 1: при `p = p`, `free()` изтрива данните, от които после `copyFrom()` трябва да копира → **crash**.

---

## 8. Деструктор

### Дефиниция

Деструкторът се извиква автоматично при унищожаване на обект — при излизане от scope или при `delete`. Отговорността му е да **освободи всички ресурси**, притежавани от обекта.

```cpp
~Person() {
    free();   // делегираме към помощната функция
}
```

### Без деструктор — изтичане на памет

```cpp
void work() {
    Person p("Иван", 20);   // заделя памет с new
}   // ← ако деструкторът не извика delete[], паметта НИКОГА не се освобождава

// При многократно извикване програмата консумира все повече памет (memory leak).
```

### Ред на извикване

Деструкторите се извикват в **обратен ред** спрямо конструкторите:

```cpp
{
    Person p1("Иван", 20);    // конструиран първи
    Person p2("Мария", 21);   // конструиран втори
}
// При излизане от scope:
// ~Person() за p2   ← унищожен първи
// ~Person() за p1   ← унищожен втори
```

## 9. Помощните функции `free()` и `copyFrom()`

При класове с динамична памет е добра практика логиката за освобождаване и копиране да се изнесе в **private помощни функции**. По този начин всяка от четирите специални функции делегира към тях — избягва се дублиране на код и намалява рискът от грешки.

```
Конструктор по подразбиране  →  инициализира указателите с nullptr
Копиращ конструктор          →  copyFrom()
Оператор=                    →  free()  +  copyFrom()
Деструктор                   →  free()
```

### `free()` — освобождаване на ресурси

```cpp
void free() {
    delete[] name;
    name = nullptr;   // критично! предотвратява double-free
}
```

`name = nullptr` след `delete[]` е критично. Ако `free()` се извика два пъти по грешка, `delete[] nullptr` е безопасно и не прави нищо. `delete[]` на произволен адрес е **недефинирано поведение**.

### `copyFrom()` — дълбоко копиране

```cpp
void copyFrom(const Person& other) {
    // Няма нужда да трием тук, понеже разчитаме, че когато трябва
    // free() функцията ще е извикана преди copyFrom()
    name = new char[strlen(other.name) + 1];
    strcpy(name, other.name);
    age = other.age;
}
```

`copyFrom()` **не извиква** `free()` сама — тази отговорност е на извикващия. Причината: копиращият конструктор няма какво да освобождава (обектът се създава за пръв път), но операторът `=` трябва да освободи старите данни преди да копира.

## 10. Пълен пример — клас `Person`

```cpp
#include <iostream>
#include <cstring>

class Person {
public:
    // --- Конструктор с параметри ---
    Person(const char* name, int age) {
        setName(name);
        setAge(age);
    }

    // --- 1. Конструктор по подразбиране ---
    Person() : name(nullptr), age(0) {}

    // --- 2. Копиращ конструктор ---
    Person(const Person& other) {
        copyFrom(other);
    }

    // --- 3. Оператор= ---
    Person& operator=(const Person& other) {
        if (this != &other) {
            free();
            copyFrom(other);
        }
        return *this;
    }

    // --- 4. Деструктор ---
    ~Person() {
        free();
    }

    // --- Setters ---
    void setName(const char* newName) {
        delete[] name;
        if (newName != nullptr) {
            name = new char[strlen(newName) + 1];
            strcpy(name, newName);
        } else {
            name = nullptr;
        }
    }

    void setAge(int newAge) {
        if (newAge >= 0 && newAge <= 120)
            age = newAge;
    }

    // --- Getters ---
    const char* getName() const { return name ? name : ""; }
    int         getAge()  const { return age; }

    void print() const {
        std::cout << "Person(" << getName() << ", " << age << ")\n";
    }

private:
    char* name = nullptr;
    int   age  = 0;

    void free() {
        delete[] name;
        name = nullptr;
    }

    void copyFrom(const Person& other) {
        name = new char[strlen(other.name) + 1];
        strcpy(name, other.name);
        age = other.age;
    }
};

int main() {
    std::cout << "=== Конструктор ===\n";
    Person p1("Иван", 20);
    p1.print();                       // Person(Иван, 20)

    std::cout << "\n=== Копиращ конструктор ===\n";
    Person p2 = p1;                   // копиращ конструктор
    p2.setName("Копие-Иван");
    p1.print();                       // Person(Иван, 20) — p1 не е засегнат!
    p2.print();                       // Person(Копие-Иван, 20)

    std::cout << "\n=== Оператор= ===\n";
    Person p3("Мария", 21);
    p3 = p1;                          // оператор=
    p3.setName("Копие-Мария");
    p1.print();                       // Person(Иван, 20) — p1 не е засегнат!
    p3.print();                       // Person(Копие-Мария, 20)

    std::cout << "\n=== Самоприсвояване ===\n";
    p1 = p1;                          // безопасно — if(this != &other) го хваща
    p1.print();                       // Person(Иван, 20)

    std::cout << "\n=== Конструктор по подразбиране ===\n";
    Person p4;
    p4.print();                       // Person(, 0)

    std::cout << "\n=== Деструктори (обратен ред) ===\n";
    // p4, p3, p2, p1 се унищожават тук автоматично
}
```

### Очакван изход

```
=== Конструктор ===
Person(Иван, 20)

=== Копиращ конструктор ===
Person(Иван, 20)
Person(Копие-Иван, 20)

=== Оператор= ===
Person(Иван, 20)
Person(Копие-Мария, 20)

=== Самоприсвояване ===
Person(Иван, 20)

=== Конструктор по подразбиране ===
Person(, 0)

=== Деструктори (обратен ред) ===
```

## 11. Edge Cases и капани

### Липса на проверка за самоприсвояване

```cpp
Person& operator=(const Person& other) {
    // ❌ БЕЗ проверка:
    free();                // изтриваме name
    copyFrom(other);       // other.name е вече изтрит ако other == *this → crash!
    return *this;
}

Person p("Иван", 20);
p = p;   // → free() изтрива "Иван" → copyFrom чете от освободена памет
```

---

### Забравено `free()` в оператор=

```cpp
Person& operator=(const Person& other) {
    if (this != &other) {
        // ❌ Забравено: free();
        copyFrom(other);   // старата памет на name изтича → memory leak
    }
    return *this;
}
```

При всяко присвояване старата памет не се освобождава. При много присвоявания паметта бавно се изчерпва.

---

### Неинициализиран указател

```cpp
class Person {
    char* name;   // ❌ стойността е произволна (боклук от стека)!
    int   age;
public:
    Person() {}   // name не е nullptr → при free(): delete[] <боклук> → crash
};
```

Указателите **винаги** трябва да се инициализират с `nullptr`:

```cpp
Person() : name(nullptr), age(0) {}   // ✅
```

---

### Двойно освобождаване (Double Free)

```cpp
void free() {
    delete[] name;
    // ❌ Забравено: name = nullptr;
}

free();   // освобождава паметта
free();   // ❌ delete[] на вече освободена памет → crash
```

Затова след `delete[]` указателят задължително се нулира: `name = nullptr`.

---

### Скрито копиране при подаване по стойност

```cpp
void printPerson(Person p) {   // ← копиращият конструктор се извиква тук!
    p.print();
}                              // ← деструкторът се извиква тук!

Person ivan("Иван", 20);
printPerson(ivan);
```

Ако копиращият конструктор не е имплементиран правилно, подаването по стойност е потенциален source на грешки. Предпочитаният подход при голям или сложен клас е подаване по `const&`:

```cpp
void printPerson(const Person& p) {   // ← без копиране, без риск
    p.print();
}
```

---

### Забравен конструктор по подразбиране

```cpp
class Person {
public:
    Person(const char* name, int age) { /* ... */ }
    // Компилаторът СПИРА да генерира Person() автоматично!
};

Person p;              // ❌ ГРЕШКА при компилация
Person arr[10];        // ❌ ГРЕШКА — масивът изисква конструктор по подразбиране
std::vector<Person> v;
v.resize(5);           // ❌ ГРЕШКА — resize изисква конструктор по подразбиране
```

## 12. Обобщение

### Голямата Четворка

```
┌──────────────────────────────────────────────────────────────┐
│                     ГОЛЯМАТА ЧЕТВОРКА                        │
├──────────────────────────┬───────────────────────────────────┤
│  Конструктор по подразб. │  T()                              │
│  Копиращ конструктор     │  T(const T& other)                │
│  Оператор=               │  T& operator=(const T& other)     │
│  Деструктор              │  ~T()                             │
└──────────────────────────┴───────────────────────────────────┘
```

### Кога се извиква кое

```
T a;              → конструктор по подразбиране
T b = a;          → КОПИРАЩ конструктор  (b се създава)
T c(a);           → КОПИРАЩ конструктор  (c се създава)
f(a);             → КОПИРАЩ конструктор  (подаване по стойност)

T d("...", 0);
d = a;            → ОПЕРАТОР=           (d вече съществуваше)
```

### Правилото — при динамична памет

> Ако класът използва `new` в конструктора, **задължително** трябва да се имплементират всичките четири функции от Голямата Четворка.

| Функция | Какво трябва да прави |
|---|---|
| Конструктор по подразбиране | Инициализира указателите с `nullptr` |
| Копиращ конструктор | Вика `copyFrom()` — **deep copy** |
| Оператор `=` | Проверка за self-assignment + `free()` + `copyFrom()` |
| Деструктор | Вика `free()` → `delete[]` |

### Структура на имплементацията

```cpp
class MyClass {
    char* data  = nullptr;
    int   size  = 0;

private:
    void free() {
        delete[] data;
        data = nullptr;
    }
    void copyFrom(const MyClass& other) {
        // no free() here — caller's responsibility
        size = other.size;
        data = new char[size];
        memcpy(data, other.data, size);
    }

public:
    MyClass()                        { /* data и size вече са nullptr/0 */ }
    MyClass(const MyClass& other)    { copyFrom(other); }
    MyClass& operator=(const MyClass& other) {
        if (this != &other) { free(); copyFrom(other); }
        return *this;
    }
    ~MyClass() { free(); }
};
```

> **Основен извод:** Автоматично генерираните функции са достатъчни само когато класът не управлява ресурси. При наличие на динамична памет те правят **shallow copy**, което води до споделяне на памет, двойно освобождаване и трудно откриваеми грешки. Решението е ръчна имплементация на всичките четири функции с **deep copy** логика, реализирана чрез помощните функции `free()` и `copyFrom()`.
