# Композиция и Агрегация в C++

**Два начина един обект да „съдържа" друг обект.**

---

## 1. Какво е връзка „има" (has-a)?

В ООП има два основни вида връзки между класове:

| Връзка | Пример | Ключова дума |
|---|---|---|
| **„е" (is-a)** | Куче **е** Животно | наследяване |
| **„има" (has-a)** | Кола **има** Двигател | композиция / агрегация |

Композицията и агрегацията са двата начина да изразим „**има**":

-  **Композиция** — частта **не може да съществува** без цялото
- **Агрегация** — частта **може да съществува** самостоятелно

---

## 2. Композиция

### Идея

> „Ако цялото умре, частта умира с него."

Класически пример: **Сърце и Човек**.  
Сърцето е създадено заедно с човека и унищожено заедно с него.  
Сърцето **няма смисъл** без човека.

### Как изглежда в C++?

При композиция обектът-част се съхранява **по стойност** вътре в класа.

```cpp
class Engine {
public:
    int horsepower;

    Engine(int hp) : horsepower(hp) {
        std::cout << "Двигател създаден (" << hp << " к.с.)\n";
    }

    ~Engine() {
        std::cout << "Двигател унищожен\n";
    }

    void start() {
        std::cout << "Двигателят работи с " << horsepower << " к.с.\n";
    }
};

class Car {
public:
    std::string brand;
    Engine engine;  // ← по СТОЙНОСТ = композиция

    Car(std::string b, int hp) : brand(b), engine(hp) {
        std::cout << "Кола " << brand << " създадена\n";
    }

    ~Car() {
        std::cout << "Кола " << brand << " унищожена\n";
    }

    void drive() {
        engine.start();
        std::cout << brand << " се движи!\n";
    }
};
```

```cpp
int main() {
    Car car("BMW", 200);
    car.drive();
}
// Изход:
// Двигател създаден (200 к.с.)
// Кола BMW създадена
// BMW се движи!
// Двигателят работи с 200 к.с.
// Кола BMW унищожена    ← колата умира
// Двигател унищожен    ← двигателят умира ЗАЕДНО с нея
```

### Ключови белези на композиция:

- Частта се декларира **по стойност** (`Engine engine;`)
- Частта се създава в **конструктора** на цялото
- Частта се унищожава **автоматично** когато цялото се унищожи
- Частта **не може да се споделя** между различни обекти

---

## 3. Агрегация

### Идея

> „Ако цялото умре, частта продължава да живее."

Класически пример: **Студент и Университет**.  
Студентът съществува независимо от университета.  
Ако университетът затвори, студентът не изчезва.

### Как изглежда в C++?

При агрегация обектът-част се съхранява като **указател или референция**.  
Обектът е създаден **отвън** и само се **подава** на класа.

```cpp
class Student {
public:
    std::string name;

    Student(std::string n) : name(n) {
        std::cout << "Студент " << name << " създаден\n";
    }

    ~Student() {
        std::cout << "Студент " << name << " унищожен\n";
    }

    void study() {
        std::cout << name << " учи\n";
    }
};

class University {
public:
    std::string name;
    Student* student;  // ← УКАЗАТЕЛ = агрегация

    University(std::string n, Student* s) : name(n), student(s) {
        std::cout << "Университет " << name << " създаден\n";
    }

    ~University() {
        std::cout << "Университет " << name << " унищожен\n";
        //  НЕ изтриваме student! Той е създаден отвън.
    }

    void teach() {
        student->study();
    }
};
```

```cpp
int main() {
    Student* ivan = new Student("Иван");  // създаден отвън

    {
        University uni("СУ", ivan);
        uni.teach();
    }  // университетът се унищожава тук

    // Иван все още съществува!
    ivan->study();  // работи!

    delete ivan;  // ние го изтриваме, не университетът
}
// Изход:
// Студент Иван създаден
// Университет СУ създаден
// Иван учи
// Университет СУ унищожен   ← университетът умира
// Иван учи                  ← Иван е жив!
// Студент Иван унищожен     ← Иван умира чак когато ние кажем
```

### Ключови белези на агрегация:

- Частта се декларира като **указател** (`Student* student;`)
- Частта е създадена **извън** класа и се **подава отвън**
- Частта **НЕ се унищожава** от деструктора на цялото
- Частта **може да се споделя** между различни обекти

---

## 4. Композиция vs Агрегация — сравнение

| | Композиция  | Агрегация  |
|---|---|---|
| **Живот на частта** | Зависи от цялото | Независим |
| **Декларация** | По стойност `Engine e;` | Указател `Engine* e;` |
| **Създаване** | Вътре в конструктора | Отвън, подава се |
| **Унищожаване** | Автоматично с цялото | Ръчно, отвън |
| **Споделяне** | Не може | Може |
| **Пример** | Сърце в Човек | Студент в Университет |
| **Пример** | Стая в Сграда | Учител в Клас |
| **Пример** | Двигател в Кола | Книга в Библиотека |

### Въпрос за бърза проверка:

> „Ако унищожа **цялото**, трябва ли **частта** да умре?"

- **Да** → Композиция
- **Не** → Агрегация

## 5. Пълен пример — Университет

Нека направим по-реалистичен пример с два класа — `Department` (Катедра) и `Professor` (Преподавател).

- Катедрата **притежава** своите стаи → **Композиция**
- Катедрата **наема** преподаватели → **Агрегация** (преподавателят съществува и извън катедрата)

```cpp
#include <iostream>
#include <string>
#include <vector>

// --- Стая (композиция с катедра) ---
class Room {
public:
    int number;
    int capacity;

    Room(int num, int cap) : number(num), capacity(cap) {
        std::cout << "  Стая " << number << " създадена\n";
    }

    ~Room() {
        std::cout << "  Стая " << number << " унищожена\n";
    }

    void info() const {
        std::cout << "  Стая " << number
                  << " (капацитет: " << capacity << " места)\n";
    }
};

// --- Преподавател (агрегация с катедра) ---
class Professor {
public:
    std::string name;
    std::string subject;

    Professor(std::string n, std::string s) : name(n), subject(s) {
        std::cout << "Проф. " << name << " създаден\n";
    }

    ~Professor() {
        std::cout << "Проф. " << name << " унищожен\n";
    }

    void teach() const {
        std::cout << "Проф. " << name << " преподава " << subject << "\n";
    }
};

// --- Катедра ---
class Department {
public:
    std::string name;

    // КОМПОЗИЦИЯ: стаите живеят вътре в катедрата
    std::vector<Room> rooms;

    // АГРЕГАЦИЯ: преподавателите са отвън — само указатели
    std::vector<Professor*> professors;

    Department(std::string n) : name(n) {
        std::cout << "Катедра " << name << " създадена\n";
    }

    ~Department() {
        std::cout << "Катедра " << name << " унищожена\n";
        //  rooms се унищожават автоматично (композиция)
        // professors НЕ се унищожават (агрегация)
    }

    void addRoom(int number, int capacity) {
        rooms.emplace_back(number, capacity);
    }

    void addProfessor(Professor* p) {
        professors.push_back(p);
    }

    void showInfo() const {
        std::cout << "\n=== Катедра: " << name << " ===\n";
        std::cout << "Стаи:\n";
        for (const auto& r : rooms)
            r.info();
        std::cout << "Преподаватели:\n";
        for (const auto* p : professors)
            std::cout << "  - Проф. " << p->name
                      << " (" << p->subject << ")\n";
    }

    void startLectures() const {
        for (const auto* p : professors)
            p->teach();
    }
};

int main() {
    // Преподавателите съществуват независимо
    Professor p1("Иванов", "C++");
    Professor p2("Петрова", "Алгоритми");

    {
        Department dept("Компютърни науки");

        // Стаи — композиция (създадени вътре)
        dept.addRoom(101, 30);
        dept.addRoom(102, 50);

        // Преподаватели — агрегация (подадени отвън)
        dept.addProfessor(&p1);
        dept.addProfessor(&p2);

        dept.showInfo();
        dept.startLectures();

    }  // ← Катедрата се унищожава тук, стаите също

    std::cout << "\n--- Катедрата е затворена ---\n";

    // Преподавателите все още съществуват!
    p1.teach();
    p2.teach();

    return 0;
}
```

### Очакван изход:

```
Проф. Иванов създаден
Проф. Петрова създадена
Катедра Компютърни науки създадена
  Стая 101 създадена
  Стая 102 създадена

=== Катедра: Компютърни науки ===
Стаи:
  Стая 101 (капацитет: 30 места)
  Стая 102 (капацитет: 50 места)
Преподаватели:
  - Проф. Иванов (C++)
  - Проф. Петрова (Алгоритми)
Проф. Иванов преподава C++
Проф. Петрова преподава Алгоритми
Катедра Компютърни науки унищожена
  Стая 101 унищожена        ← стаите умират с катедрата
  Стая 102 унищожена

--- Катедрата е затворена ---
Проф. Иванов преподава C++     ← преподавателите са живи!
Проф. Петрова преподава Алгоритми
Проф. Петрова унищожена
Проф. Иванов унищожен
```

## 6. Чести грешки

### Грешка 1: Изтриване на агрегирания обект в деструктора

```cpp
~University() {
    delete student;  // ❌ ГРЕШКА! Ние не притежаваме student-а!
                     // Той е създаден отвън и ще се изтрие два пъти
}
```

**Решение:** В агрегация деструкторът НЕ изтрива указателя.

### Грешка 2: Dangling pointer при агрегация

```cpp
University* uni;

{
    Student ivan("Иван");
    uni = new University("СУ", &ivan);
}  // ivan е унищожен тук!

uni->teach();  // ❌ ГРЕШКА! uni->student сочи към унищожен обект!
```

**Решение:** Увери се, че обектът-агрегат живее **поне толкова дълго** колкото обектът, който го използва.

###  Грешка 3: Объркване на двете

```cpp
class Car {
    Engine* engine;  // указател — изглежда като агрегация...

    Car(int hp) {
        engine = new Engine(hp);  // ...но го създаваме вътре!
    }

    ~Car() {
        delete engine;  // ...и го унищожаваме вътре = всъщност е КОМПОЗИЦИЯ
    }
};
```

Това е технически **композиция** реализирана с указател. По-добре е да използваш `std::unique_ptr<Engine>` в такъв случай ( когато го учим ).
